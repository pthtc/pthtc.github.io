概述

概念

  • 信息： 事物运动的状态及其状态变化的方式
  • 媒体： 在不同对象中实现互相沟通
  • 脆弱点： 为安全事件发生提供了条件
  • 安全需求： 保密、安全、可用
  • 核心要素： 人、技术、操作

五级体系结构

物理层 网线等，与链路层组成局域网

链路层 交换机、路由器等发挥作用的地方

网络层 IP

运输层 TCP/UDP

应用层 DNS/FTP/HTTP/URL/SMTP/POP3/IMAP等，交换的数据单元叫报文

新信息安全防护的原则

  1. 整体性： 从整体上构思、设计整体安全框架（木桶原理）
  2. 分层性： 不依赖单一机制，保证存在荣誉，增加安全层次

粉碎棱镜： 加密

设备与环境安全

问题

  1. 环境事故造成的设备故障： 天灾、灰尘、电磁干扰等
  2. 硬件缺乏安全防护： 机箱拆卸、内存条读取等
  3. 硬件恶意代码
  4. 旁路攻击： 分析键盘声音、cpu电磁波等
  5. 在线威胁： 设备连接互联网后在黑客攻击范围内

物理防护

  1. 环境安全
  2. 电磁安全
  3. PC物理防护： 锁扣等
  4. 设备访问控制： 保护文件不被非法访问（安全芯片、软件狗）

数据安全

散列函数和数据完整性

散列函数（哈希）

  1. 只有加密，没有解密
  2. 不可预见
  3. 任意大小信息
  4. 抗碰撞

  5. h = H(M)

  6. 校验数据完整性、重复数据

  7. 消息认证，避免伪装
  8. 数字签名
  9. 保护用户口令

MD5已经被碰撞攻破

数字签名与数据不可否认性和可认证性

数字签名

用于确保当事人身份真实和信息的不可否认

秘钥分发中心： 双方均认可实体

  1. 不可否认
  2. 不可伪造
  3. 可认证

基于公钥的数字签名

  1. 发送者用私钥对明文加密为密文，形成数字签名
  2. 同时发送密文和公钥
  3. 接收者用公钥解密得到明文

基于散列函数和公钥的数字签名

  1. 发送者对明文计算散列值，再用私钥对散列值进行数字签名
  2. 发送明文和签名和公钥
  3. 对签名解密，获得散列值，同时用明文计算散列值，相等则成功

身份与访问安全

身份认证技术

  1. 利用用户所知道的认知： 口令、图片
  2. 利用用户所拥有的认证： 密保卡、动态口令牌、USB KEY
  3. 利用用户本身的特诊认证： 指纹、虹膜、人脸、生物行为（按键、鼠标、步态等）

身份认证机制

  1. 一次性口令： 验证码、动态密码
  2. 可信第三方： PKI（https）

PKI公钥基础设施

围绕数字证书产生，可根据数字签名判断证书出处

  • 注册授权中心RA： 用户和CA的中间人，对于第一次使用PKI的用户，负责建立和认证用户身份，将请求转发给CA
  • 认证授权中心CA： 存储管理发放数字证书
  • 数字证书库：存储数字证书

通讯过程:

  1. A获取B证书，提取B公钥
  2. 使用B公钥加密key，key用于加密通讯内容
  3. 发送加密后的key和公钥A
  4. B收到之后，验证A证书是否来自可信CA
  5. B用私钥解密，获得key，通讯开始

特点：

  1. 可认证性
  2. 不可抵赖性
  3. 机密性
  4. 数据完整性

信任模型

独立PKI之间的互信

  1. 层次模型： 上级是下级的信任锚
  2. 交叉模型
  3. 网状模型
  4. 混合模型

访问控制模型

  1. 基本访问控制模型： 将所有访问控制信息储存在一个矩阵(表)中进行集中管理
  2. 自主访问控制模型： 由课题的属主对自己的客体进行管理，由属主决定是否将自己的客体访问权或部分访问权授予其他主体
  3. 强制访问控制模型： 基于安全标签，安全级分为保密级别（机密程度）和范畴级（所属部门），不能向上读（用于保密），不能向下写（用于防范木马）

系统软件安全

安全等级 CC

CC: EAL1~EAL7

TCSEC: D、C1、C2、B1、B2、B3、A1

安全目标

  1. 身份
  2. 访问控制
  3. 监督
  4. 自身安全和完整

机制

  1. 身份鉴别，合法用户才能存取资源
  2. 访问控制，凭借身份做事
  3. 最小权限管理，只给够做事的权限
  4. 信道保护，确认是和核心通信而不是伪装程序，隐蔽信道（违反安全策略的信道）处理
  5. 审计，对系统有关圈的活动进行记录、检查和审核
  6. 存储保护，内存、运行、IO保护
  7. 文件系统保护，分区、文件系统加载、文件共享、数据备份保护

网络系统安全

黑客与网络攻击

攻击过程

  1. 隐藏攻击源（肉鸡、多级代理、伪造IP、假冒账号）
  2. 信息搜集（确定攻击目标、收集目标信息、扫描漏洞和平台类型、嗅探获取敏感信息）
  3. 掌握系统控制权（口令猜测、木马、会话劫持）
  4. 实施攻击（修改获取敏感信息、攻击其他被信任的主机、瘫痪网络服务、其他）
  5. 安装后门以保持长期控制（放宽系统许可权、重新开放不安全服务、修改系统配置、替换共享库文件、安装木马、修改源代码）
  6. 清理攻击哼唧（清除篡改日志、改变系统时间造成日志数据紊乱、利用调班隐藏真实攻击者）

APT

APT： 高级持续性威胁，利用先进的攻击手段对特定目标进行长期持续性网络攻击的攻击形式，威胁大

和传统比较

  1. 攻击者有组织、资金背景强————普通个人、组织
  2. 目标为国家重要设施、人员————大范围搜索
  3. 数据为价值很高的电子资产————信用卡数据、个人资料
  4. 目的为提升国家战略优势、摧毁关键设施————经济利益
  5. 手段为深入调查————传统技术手段
  6. 针对目标漏洞定制工具————常用工具
  7. 常用0day漏洞
  8. 遇到阻力后构造新方法————放弃

网络安全设备

防火墙

一种隔离器，分析器

  1. 唯一出入口
  2. 自身抗攻击，根据安全策略控制信息流
  3. 不能影响网络信息流通

包过滤技术

根据包特征决定是否丢弃

  1. 静态包过滤，提前定义好规则，根据包头信息，分为严策略和宽策略
  2. 状态包过滤，同时维护规则表和状态表，状态表保存当前活动的合法连接，新的连接通过验证添加，连接完成删除

应用代理技术

应用层网关技术

  1. 双宿主网关: 有一台堡垒主机（两块网卡）作为防火墙，连接内外网
  2. 屏蔽主机网关： 堡垒主机前又多了一台屏蔽路由器作为防火墙（只允许堡垒主机和外网访问）
  3. 屏蔽子网防火墙： 将堡垒主机、web、邮件、公共服务器放在被隔离子网（DMZ），对外对内各有一个屏蔽路由器

IPSEC网络层安全协议

IPV6 组成部分，IPV4可选扩展

  1. 传输模式：用于两台主机间端到端通信，发送端IPSEC将包内容加密认证，IP头不认证，接收端IPSEC解密
  2. 隧道模式： 点对点通信，整包加密

包含：

  1. 认证头，用于数据完整性和IP包认证
  2. 载荷安全封装，确保完整和机密、验证
  3. 因特网秘钥交换，IPSEC之间建立共享安全参数及验证的秘钥

应用软件安全

恶意代码

未授权情况下以破坏、窃取、干扰编写的软件或代码片段，有木马、蠕虫（自传播）、病毒（破坏）等

代码安全漏洞

脆弱性，计算机的缺陷和意想不到的问题

  1. 缓冲区一出漏洞，通过在缓冲区写入超长内容，破坏堆栈、使程序转而执行其它指令
  2. sql注入漏洞，输入的查询字符串中注入sql语句，会被误认为是正常sql指令执行，造成数据泄露
  3. 心脏出血漏洞，openssl，没有对信条数据包进行边界检测，都指heartbeat模块会将明文数据直接输出
